-标题：基于 DDD 的 TypeScript 支付系统设计（Koa + 微信/支付宝 + Docker + 消息队列 + 含对账周期 + 系统化教学）
- 背景与目标
  - 我是 DDD 零基础，希望边学边做，最终产出可运行的多支付平台支付系统（微信/支付宝），并清晰解释“为什么这样设计”（领域边界、上下文、领域模型、分层架构、平台适配等）。
- 业务范围
  - 内：创建支付、确认/扣款、取消、退款、异步回调（Webhook）、支付状态查询、重试与幂等、平台账单对账（含对账周期与差异处理）。
  - 外：订阅、分账、优惠券、风控（如未来需要，列为后续迭代）。
- 非功能性要求
  - 幂等：入口 `Idempotency-Key`、回调去重、出站 Outbox。
  - 一致性：最终一致性（Outbox/Saga 思路），失败补偿与重试（指数退避、最大次数）。
  - 可观测性：结构化日志、追踪 ID、基础指标（成功率/失败率/耗时/重试次数）。
  - 安全：签名校验、防重放、证书与密钥管理、日志脱敏、审计留痕、最小权限访问。
- 技术栈与框架（Koa + 默认库）
  - Node.js 20+、TypeScript 5+；Web 框架：Koa；路由：`@koa/router`；校验：`zod`；依赖注入：`awilix`。
  - ORM：Prisma；数据库：PostgreSQL；消息系统：RabbitMQ；容器：Docker/Docker Compose（PostgreSQL + RabbitMQ + 应用）。
  - 测试：Jest/Vitest + `supertest`；文档：OpenAPI（可通过注解或手写）。
- Koa 专项约束
  - 中间件顺序：请求 ID → 日志 → 错误处理 → 安全/限流 → 解析器 → 鉴权 → 幂等键 → 路由。
  - Webhook 路由需保留原始请求体以做签名校验（与通用 JSON 解析路由分离）。
  - 统一错误响应规范（校验错误/业务错误/系统错误）与日志脱敏策略。
- DDD 教学与设计要求
  - 用通俗中文讲清：实体、值对象、聚合、领域服务、领域事件、限界上下文、统一语言。
  - 解释如何划分边界、上下文映射及与一致性边界/团队边界/业务语言的关系。
  - 分层架构：接口层（Koa 控制器/DTO/校验）、应用层（用例/事务编排）、领域层（纯模型）、基础设施层（仓储/外部适配器/消息/HTTP）。
- 多支付平台对接（微信/支付宝）
  - 端口-适配器模式 + 防腐层（ACL），屏蔽平台差异；新增平台仅新增适配器。
  - 签名与证书：微信 v3（平台证书轮换、时间戳与随机串校验）；支付宝 RSA2（开放平台公钥校验）。
  - 回调：严格原文验签、去重（事件唯一键）、重试策略（幂等）。
  - 默认支付场景（如未另行说明）：微信 Native/H5，支付宝 PC 网页/手机网页；扫码/跳转收银台为主。
- 对账设计（已纳入范围）
  - 周期：T+1 每日对账，日切 00:00 Asia/Shanghai。
  - 数据源：拉取微信/支付宝对账单（账务/资金），解析并与内部交易流水比对。
  - 匹配与差异：按商户订单号/平台交易号匹配；差异分类（缺失、金额不符、状态不一致）；生成对账结果与调账建议。
  - 结果沉淀：生成对账报表与指标；支持重跑与异常重试。
- 交付顺序（严格按序）
  1) DDD 基础教学与术语表（中文）
  2) 领域划分与上下文映射（支付/回调/对账的边界与通信）
  3) 核心用例与状态机/时序图
  4) 分层架构与目录结构（Koa 接口层如何落地）
  5) 领域模型与接口清单（只讲设计，不写实现）
  6) 适配器与集成策略（微信/支付宝签名、幂等、防腐层）
  7) 最小可运行骨架与关键用例演示（Koa 路由/中间件链路）
  8) 文档与运行手册（README、C4/ADR、统一语言表、对账说明）
- 验收标准
  - 本地（Docker）运行 Demo：发起支付 → 接收模拟 Webhook（原始报文验签） → 状态更新 → 可发起退款 → 生成对账结果；全程有日志与基础测试。
  - 新增平台无需修改应用/领域层，仅新增适配器即可通过验收。
- 安全/合规要求（默认基线）
  - 无 PCI DSS 强依赖（不自持卡数据），但需：
    - 密钥与证书安全：采用 KMS/封装的密钥管理、定期轮换；微信平台证书自动更新，支付宝开放平台公钥管理。
    - 传输安全：TLS 1.2+；Webhook 域名白名单与签名校验；防重放（时间戳窗口、一次性 nonce）。
    - 日志与隐私：PII/敏感字段脱敏；签名参数/密文不落日志；访问审计。
    - 配置与权限：最小权限、环境变量管理（.env 分层）、只读服务账号。
- 已确认的边界条件
  - 非多货币、非多时区、非多租户；时区默认 Asia/Shanghai。
  - 默认库与组件：`@koa/router` + `zod` + `awilix` + Prisma + PostgreSQL + RabbitMQ + Docker Compose。
- 运维与运行
  - 提供 `docker-compose.yml`（App + PostgreSQL + RabbitMQ），健康检查、就绪探针、基础监控指标暴露。
  - 本地/测试环境使用微信/支付宝沙箱；提供环境变量模板与回调 URL 配置说明。
